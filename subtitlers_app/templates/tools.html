Для перекладу субтитрів «під клацання» слід використовувати не текстовий процесор, а простий текстовий редактор, — наприклад, Atom, Visual Studio Code чи Sublime Text.

Переваги

  • Вони дозволяють створювати текстові файли (.txt або .srt) без зайвого форматування (наприклад, не інтерпретують дефіс із пробілом як елемент списку і не зроблять з трьох крапок один символ).
  • При виділенні фрагмента тексту ви одразу бачите кількість символів.
  • Ви можете налаштувати редактор так, щоб символи, які виходять за межі допустимої кількості, автоматично переносилися на новий рядок (див. скріншот нижче).
  • Рядки одразу пронумеровані (зручно, коли треба порахувати кількість субтитрів).
  • Простий дизайн і мінімальний функціонал не відволікають від роботи :)
  • Якщо порівнювати з Google Docs — ваш комп не гальмуватиме за великих обсягів тексту :)

Якщо ви одержуєте в роботу файл англійських субтитрів із готовими таймкодами, необхідно працювати в одній із субтитрувальних програм, наприклад, Subtitle Edit чи Subtitle Workshop (для Windows) або Aegisub (для macOS). Це потрібно для того, щоб у процесі роботи ненароком не зіпсувати таймкод.

При роботі над такими файлами в звичайному блокноті чи іншому текстовому редакторі, таймкоди дуже часто ламаються. Виявлення і виправлення помилок займає багато часу (в той час, коли його зазвичай немає).

Субтитри «проклацуються» в програмі subtitler 9 (Windows).

Таблиці

З таблицями зазвичай просто — потрібно скопіювати текст, вставити його у свій текстовий редактор і замінити табуляцію на пробіли. Табуляцію можна виявити за допомогою regular expressions — їй відповідає символ \t. Дивіться скріншот нижче.

Колишні клітинки таблиці розділені за допомогою табуляції (Ctrl/Cmd + F для пошуку, зірочкою (*) уможливлюєте regular expressions, вводите \t у полі пошуку):

Конвертація

Програми для роботи з субтитрами відкривають і конвертують більшість форматів субтитрів без проблем. Також пам’ятайте, що будь-який текстовий файл ви можете відкрити у своєму текстовому редакторі (саме редакторі, а не процесорі). Якщо файл бінарний (не текстовий), подивіться, яка програма може його відкрити. В крайньому разі — запитайте у координаторів перекладу або скористайтеся програмами-конвертерами файлів (за запитом [X] to [Y] file converter).

Наступні інструменти дозволяють:

  • перевірити, чи трапляються 3-рядкові субтитри,
  • Space Remover — замінити подвійні пробіли одинарними,
  • Timecode Checker — перевірити послідовність таймкодів.
